Transformación de Datos

Introducción

La transformación de datos es el proceso de convertir datos de un formato o estructura a otro. Es esencial para preparar los datos para el análisis y modelado.

1. Normalización y Estandarización

Estas técnicas escalan las características numéricas a un rango común sin distorsionar las diferencias en los rangos de valores.

Min-Max Normalization (Normalización Min-Max)

Escala los valores al rango [0, 1]:

X_norm = (X - X_min) / (X_max - X_min)
  • Ventajas: Preserva la forma de la distribución original
  • Desventajas: Sensible a outliers
  • Cuándo usar: Cuando conoces los límites de tus datos

Standardization (Estandarización Z-score)

Transforma los datos para tener media 0 y desviación estándar 1:

X_std = (X - μ) / σ
  • Ventajas: Menos sensible a outliers que Min-Max
  • Desventajas: No garantiza un rango específico
  • Cuándo usar: Para algoritmos que asumen distribución normal

Robust Scaling (Escalado Robusto)

Usa la mediana y el rango intercuartílico (IQR):

X_robust = (X - median) / IQR
  • Ventajas: Muy robusto a outliers
  • Cuándo usar: Cuando tienes muchos outliers

MaxAbs Scaling

Escala dividiendo por el valor absoluto máximo:

X_maxabs = X / |X_max|
  • Ventajas: No desplaza/centra los datos
  • Cuándo usar: Para datos dispersos (sparse data)

2. Codificación de Variables Categóricas

Los algoritmos de machine learning requieren datos numéricos. Debemos convertir variables categóricas a formato numérico.

One-Hot Encoding (Codificación One-Hot)

Crea una columna binaria para cada categoría:

Original Color_Rojo Color_Verde Color_Azul
Rojo 1 0 0
Verde 0 1 0
Azul 0 0 1
  • Ventajas: No asume orden entre categorías
  • Desventajas: Aumenta dimensionalidad
  • Cuándo usar: Variables nominales sin orden

Label Encoding (Codificación de Etiquetas)

Asigna un número entero a cada categoría:

Original Codificado
Rojo 0
Verde 1
Azul 2
  • Ventajas: Simple, no aumenta dimensionalidad
  • Desventajas: Implica orden artificial
  • Cuándo usar: Variables target en clasificación

Ordinal Encoding (Codificación Ordinal)

Similar a Label Encoding pero respeta el orden natural:

Original Codificado
Bajo 0
Medio 1
Alto 2
  • Cuándo usar: Variables ordinales con orden natural

Target Encoding (Codificación por Target)

Reemplaza cada categoría con la media del target para esa categoría:

  • Ventajas: Captura relación con el target
  • Desventajas: Riesgo de overfitting
  • Cuándo usar: Alta cardinalidad, con validación cruzada

3. Transformaciones de Características

Transformación Logarítmica

Útil para datos con distribución sesgada:

X_log = log(X + 1)

Transformación de Potencia (Box-Cox)

Encuentra la mejor transformación de potencia para normalizar datos.

Binning (Discretización)

Convierte variables continuas en categóricas:

  • Edad → Grupos de edad (0-18, 19-35, 36-60, 60+)
  • Ingreso → Rangos de ingreso
Importante

Siempre aplica las mismas transformaciones a los datos de entrenamiento y prueba. Guarda los parámetros de transformación (media, desviación, etc.) del conjunto de entrenamiento y úsalos para transformar el conjunto de prueba.

Mejores Prácticas
  • Escala después de dividir train/test para evitar data leakage
  • Usa One-Hot para variables nominales, Ordinal para ordinales
  • Considera la interpretabilidad del modelo al elegir codificación
  • Documenta todas las transformaciones aplicadas
