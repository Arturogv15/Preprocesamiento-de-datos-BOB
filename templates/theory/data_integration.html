Integración de Datos

Introducción

La integración de datos es el proceso de combinar datos de diferentes fuentes en una vista unificada. Es fundamental cuando trabajamos con múltiples datasets relacionados.

1. Tipos de Joins (Uniones)

Los joins combinan filas de dos o más tablas basándose en una columna relacionada.

Inner Join (Unión Interna)

Retorna solo las filas que tienen coincidencias en ambas tablas.

Ejemplo:

Tabla A: Clientes (ID, Nombre)

Tabla B: Pedidos (ID_Cliente, Producto)

Resultado: Solo clientes que han hecho pedidos

  • Cuándo usar: Cuando solo necesitas registros que existen en ambas tablas
  • Resultado: Menor o igual número de filas que las tablas originales

Left Join (Unión Izquierda)

Retorna todas las filas de la tabla izquierda y las coincidencias de la derecha.

Ejemplo:

Resultado: Todos los clientes, con sus pedidos si existen (NULL si no)

  • Cuándo usar: Cuando quieres mantener todos los registros de la tabla principal
  • Resultado: Al menos tantas filas como la tabla izquierda

Right Join (Unión Derecha)

Retorna todas las filas de la tabla derecha y las coincidencias de la izquierda.

  • Cuándo usar: Similar a Left Join pero priorizando la tabla derecha
  • Nota: Menos común, generalmente se usa Left Join cambiando el orden

Outer Join (Unión Externa)

Retorna todas las filas de ambas tablas, con NULL donde no hay coincidencias.

Ejemplo:

Resultado: Todos los clientes Y todos los pedidos, incluso si no coinciden

  • Cuándo usar: Cuando necesitas ver todos los datos de ambas tablas
  • Resultado: Máximo número de filas

Cross Join (Producto Cartesiano)

Combina cada fila de la primera tabla con cada fila de la segunda.

  • Cuándo usar: Raramente, para generar todas las combinaciones posibles
  • Cuidado: Puede generar datasets muy grandes (n × m filas)

2. Concatenación

Une datasets apilándolos vertical u horizontalmente.

Concatenación Vertical (axis=0)

Apila datasets uno debajo del otro:

df_combined = pd.concat([df1, df2], axis=0)
  • Requisito: Mismas columnas
  • Uso: Combinar datos del mismo tipo de diferentes fuentes

Concatenación Horizontal (axis=1)

Apila datasets lado a lado:

df_combined = pd.concat([df1, df2], axis=1)
  • Requisito: Mismo número de filas o índices compatibles
  • Uso: Agregar nuevas características

3. Merge (Fusión)

Combina datasets basándose en columnas clave comunes.

Sintaxis Básica

pd.merge(df1, df2, on='columna_clave', how='inner')

Parámetros Importantes

  • on: Columna(s) para hacer el join
  • left_on, right_on: Cuando las columnas tienen nombres diferentes
  • how: Tipo de join ('inner', 'left', 'right', 'outer')
  • suffixes: Sufijos para columnas duplicadas

4. Manejo de Conflictos

Columnas Duplicadas

Cuando ambas tablas tienen columnas con el mismo nombre:

pd.merge(df1, df2, on='id', suffixes=('_left', '_right'))

Valores Faltantes

Los joins pueden introducir valores NULL:

  • Left/Right/Outer joins crean NULL donde no hay coincidencias
  • Decide si mantener, eliminar o imputar estos valores

Duplicados

Si hay múltiples coincidencias, se crean múltiples filas:

  • Verifica duplicados antes del merge
  • Considera usar drop_duplicates() si es necesario

5. Mejores Prácticas

Antes de Integrar
  • Verifica la calidad de las columnas clave
  • Asegúrate de que los tipos de datos coincidan
  • Identifica y maneja duplicados
  • Documenta la lógica de integración
Cuidado Con
  • Explosión de datos: Joins pueden multiplicar filas
  • Pérdida de datos: Inner joins pueden eliminar información
  • Memoria: Cross joins pueden ser muy grandes
  • Performance: Joins en datasets grandes son costosos

6. Casos de Uso Comunes

Enriquecimiento de Datos

Agregar información adicional de otra fuente:

  • Transacciones + Información de clientes
  • Productos + Categorías

Consolidación

Combinar datos de múltiples períodos o regiones:

  • Ventas de diferentes meses
  • Datos de diferentes sucursales

Análisis Relacional

Explorar relaciones entre entidades:

  • Clientes y sus compras
  • Estudiantes y sus calificaciones
Consejo Pro

Siempre verifica el número de filas antes y después del merge. Un cambio inesperado puede indicar problemas con las claves o duplicados.
