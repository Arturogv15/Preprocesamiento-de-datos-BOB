Reducción de Dimensionalidad

Introducción

La reducción de dimensionalidad es el proceso de reducir el número de variables (características) en un dataset mientras se retiene la mayor cantidad de información posible.

¿Por qué Reducir Dimensionalidad?

  • Maldición de la dimensionalidad: Muchas características pueden degradar el rendimiento
  • Visualización: Difícil visualizar datos en más de 3 dimensiones
  • Velocidad: Menos características = entrenamiento más rápido
  • Overfitting: Reducir características puede prevenir sobreajuste
  • Ruido: Eliminar características irrelevantes mejora la señal

1. PCA (Principal Component Analysis)

PCA es una técnica de extracción de características que transforma las características originales en un nuevo conjunto de características no correlacionadas llamadas componentes principales.

¿Cómo Funciona PCA?

  1. Estandarización: Escala las características
  2. Matriz de Covarianza: Calcula cómo varían las características juntas
  3. Eigenvectores y Eigenvalores: Encuentra las direcciones de máxima varianza
  4. Selección de Componentes: Elige los k componentes principales
  5. Transformación: Proyecta los datos en el nuevo espacio

Varianza Explicada

Cada componente principal explica un porcentaje de la varianza total:

Ejemplo:

  • PC1: 45% de varianza
  • PC2: 30% de varianza
  • PC3: 15% de varianza
  • PC4: 10% de varianza

Con PC1 y PC2 capturas 75% de la información original

Scree Plot

Gráfica que muestra la varianza explicada por cada componente. Ayuda a decidir cuántos componentes mantener.

Ventajas de PCA

  • Reduce dimensionalidad manteniendo información
  • Elimina multicolinealidad
  • Mejora visualización
  • Puede mejorar rendimiento de modelos

Desventajas de PCA

  • Los componentes son difíciles de interpretar
  • Asume relaciones lineales
  • Sensible a escala (requiere estandarización)
  • Puede perder información importante

Cuándo Usar PCA

  • Tienes muchas características correlacionadas
  • Necesitas visualizar datos de alta dimensión
  • Quieres reducir ruido
  • La interpretabilidad no es crítica

2. Selección de Características

A diferencia de PCA que crea nuevas características, la selección elige un subconjunto de las características originales.

Métodos de Selección

a) Filter Methods (Métodos de Filtro)

Evalúan características independientemente del modelo:

Variance Threshold (Umbral de Varianza)
  • Elimina características con baja varianza
  • Características constantes o casi constantes no aportan información
  • Ejemplo: Eliminar características con varianza < 0.01
Correlation-based (Basado en Correlación)
  • Selecciona características más correlacionadas con el target
  • Elimina características altamente correlacionadas entre sí
  • Umbral común: |correlación| > 0.8 indica redundancia
Statistical Tests (Pruebas Estadísticas)
  • Chi-cuadrado: Para características categóricas
  • ANOVA F-test: Para características numéricas
  • Mutual Information: Captura relaciones no lineales

b) Wrapper Methods (Métodos Envolventes)

Evalúan subconjuntos de características usando un modelo:

Recursive Feature Elimination (RFE)
  1. Entrena modelo con todas las características
  2. Elimina la característica menos importante
  3. Repite hasta alcanzar el número deseado
  • Ventaja: Considera interacciones entre características
  • Desventaja: Computacionalmente costoso
Forward/Backward Selection
  • Forward: Comienza sin características, agrega una a la vez
  • Backward: Comienza con todas, elimina una a la vez

c) Embedded Methods (Métodos Embebidos)

La selección ocurre durante el entrenamiento del modelo:

Feature Importance (Importancia de Características)
  • Árboles de decisión y Random Forests calculan importancia
  • Selecciona las k características más importantes
  • Rápido y efectivo
L1 Regularization (Lasso)
  • Penaliza coeficientes, algunos se vuelven cero
  • Características con coeficiente cero se eliminan
  • Selección automática durante entrenamiento

3. Comparación: PCA vs Selección

Aspecto PCA Selección de Características
Interpretabilidad Baja (nuevas características) Alta (características originales)
Información Retiene máxima varianza Puede perder información
Velocidad Rápido Variable (RFE es lento)
Uso Visualización, preprocesamiento Interpretación, simplicidad

4. Mejores Prácticas

Recomendaciones
  • Estandariza antes de PCA: PCA es sensible a escala
  • Usa Scree Plot: Para decidir número de componentes
  • Mantén 80-95% de varianza: Balance entre reducción e información
  • Prueba múltiples métodos: Compara resultados
  • Valida con cross-validation: Evita overfitting
Cuidado
  • No uses PCA antes de dividir train/test: Data leakage
  • PCA no siempre mejora: Prueba con y sin reducción
  • Considera el dominio: Algunas características pueden ser importantes aunque tengan baja varianza

5. Casos de Uso

Usar PCA cuando:

  • Tienes muchas características correlacionadas
  • Necesitas visualizar datos (reducir a 2-3 dimensiones)
  • Quieres eliminar ruido
  • La interpretabilidad no es crítica

Usar Selección cuando:

  • Necesitas interpretar el modelo
  • Quieres identificar características importantes
  • Tienes características con significado del dominio
  • El dataset no es muy grande
Tip Profesional

Comienza con selección de características para entender tus datos. Luego, si necesitas más reducción, aplica PCA. Esta combinación puede dar los mejores resultados.
